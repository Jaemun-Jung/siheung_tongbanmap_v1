# 다음 작업 메모 (시흥시 통반경계도)

> 전체 동 데이터 준비(20개 행정동 CSV·config·산출물)까지 완료된 상태에서 남은 보완 작업.

## 1. (우선) 공유 법정동 과대확장 — 경계 제한 매칭

### 증상
정왕 지역 일부 동에서 매칭 필지가 **자기 행정동 경계 밖**까지 흡수됨.
빌드 검증 시 "행정동 내부비율"로 확인됨:

| 행정동 | 내부비율 | 경계밖 필지 |
|---|---|---|
| 정왕3동(31150620) | 63.8% | 1112 |
| 정왕본동(31150640) | 93.9% | 520 |
| 정왕4동(31150720) | 90.6% | 5 |
| (정상 14개 동) | 99.1~100% | 0~37 |

### 원인 (코드 오류 아님)
**정왕동은 단일 법정동(4139013200, 13,121필지)이 6개 행정동(정왕본/1/2/3/4동·거북섬동)에 공유**됨.
각 정왕 행정동 config의 `target_beopjeong=["정왕동"]` → 정왕동 전체를 대상으로 삼고,
특히 **본번폴백(BONBUN_FALLBACK)이 본번의 모든 부번을 같은 통에 채우면서 이웃 정왕동(다른 행정동)까지 흡수**.

### 수정 방법 (빌드 로직, 데이터 변경 없음 / 약 5줄)
`build_tongban_map.py` `main()`의 **3) 대상 추출 직후, 4) 조인 전**에 대상 필지를 행정동 경계로 제한:
```python
if CLIP_TO_ADMIN:
    gp = load_admin_outline().to_crs(sub.crs).union_all()
    sub = sub[sub.representative_point().within(gp)].copy()   # 경계 밖 필지 제외
```
- `load_admin_outline()`는 이미 존재(현재 5단계에서 외곽선·내부비율에 사용) → 호출만 앞으로.
- 대상이 경계 안으로 좁혀지면 본번폴백 과대확장도 자동 해결.
- `CLIP_TO_ADMIN`은 `apply_config`에서 config `clip_to_admin`로 읽기(이때 동시에 추가).

### 군자동 골든 주의
경계 제한을 켜면 군자동에서 경계 밖 2필지가 빠져 **동일 재현이 2필지 달라짐**.
→ **config 플래그로 동별 on/off** 권장: 군자동 `false`(골든 보존), 정왕/배곧 `true`.
(또는 군자동 골든을 재기준화.)

### 검증
수정 후 정왕3동/정왕본동 재빌드 → 내부비율 ~100%, 경계밖 필지 ≈0 확인.
14개 정상 동은 `clip_to_admin:false`로 두면 무변동.

## 2. (구조적 한계) 간척 신도시 저커버리지
- **배곧동(4139013500)은 지적상 564필지뿐**(미분할 대형 블록). 35만 인구를 못 그림.
- 정왕2/4동, 거북섬동도 아파트형 반 비중이 커 지번 매칭이 적음(기획서의 '아파트형 반 38% 한계'가 신도시에서 증폭).
- 경계 제한(1번)으로 과대확장은 잡히지만 **커버리지 자체는 건물/동·호 데이터 없이는 한계**. 별도 과제.

## 참고: 전체 동 데이터 파이프라인
1. `python extract_hwpx_tables.py "<별표.hwpx>" --out data --skip-existing` → 동별 관할구역.csv
2. `python derive_legal_codes.py` → data/siheung_legal_codes.json (법정동명↔코드, 공간검증)
3. `python gen_configs.py` → 동별 config.json (군자동 보존)
4. `python build_tongban_map.py --config data/{코드}/config.json` → 동별 산출물
- 행정동코드: data/siheung_admin_codes.json (행정동경계 ADM_CD)
