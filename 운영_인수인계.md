## 1. 이 시스템이 뭔가
시흥시 **20개 행정동**의 통·반을 조회하는 웹 지도. 「시흥시 통·반 설치 조례 시행규칙 **별표**」(통·반 관할구역)의 지번을 **연속지적도** 필지에 매칭해 경계를 자동 생성. 주소·지번·아파트·통으로 검색 → 지도 표시. **단일 파일 앱(`index.html`)** + `out/`의 GeoJSON/인덱스를 fetch.

## 2. 어디에 뭐가 있나 (아키텍처)
```
index.html            ← 앱 전체(Leaflet 셸 + UI + 검색). 키 상수 포함(아래 5번)
build_tongban_map.py  ← 별표→지적도 매칭 빌드(핵심). 동 단위 실행
gen_*.py              ← 파생 데이터 생성기(아래 3번 파이프라인)
data/{행정동코드}/
  관할구역.csv         ← ★별표(통·반 관할 지번). 데이터 수정은 보통 여기
  config.json         ← 동별 설정(법정동 매핑 등)
out/{행정동코드}/*.geojson   ← 빌드 산출물(통·필지·미배정 등)
out/*.json            ← 검색·이슈 인덱스(address_index, issues, manifest, unassigned_index ...)
out/overview/         ← 전체지도용(all_tong, all_admin, city_boundary)
LSMD_CONT_LDREG_41390_202606.shp  ← 연속지적도 원본(★gitignore. 저장소엔 없음)
.venv/                ← 파이썬 가상환경(★gitignore)
```
행정동코드 예: 군자동 `31150680`, 대야동 `31150510`, 거북섬동 `31150760`, 신천동 `31150520` … (`data/` 폴더명이 곧 코드)

## 3. ★가장 중요 — 데이터(별표) 수정 → 지도 반영 파이프라인
예: "○○동 어떤 지번의 통을 고쳐야 한다"를 반영하는 표준 절차.

**① 소스 수정** — `data/{코드}/관할구역.csv`의 해당 줄을 고침 (형식: `법정동,통,반,"지번 나열"`, 범위는 `96～105`, 나열은 `1ㆍ2ㆍ3`).

**② 그 동만 재빌드**
```powershell
cd "C:\Users\김정순\OneDrive\바탕 화면\shiheung_tongban_map\siheung-gunja-tongban-map"
$env:PYTHONUTF8=1
.venv\Scripts\python.exe build_tongban_map.py --config data\{코드}\config.json --shp LSMD_CONT_LDREG_41390_202606.shp
```

**③ 파생 데이터 재생성**(전 동을 훑지만, 바뀐 동만 달라지고 나머지는 동일 재현)
```powershell
.venv\Scripts\python.exe gen_unassigned.py          # 미배정 대지(지적도 SHP 필요, 제일 느림)
.venv\Scripts\python.exe gen_unassigned_index.py    # 미배정 위치 인덱스
.venv\Scripts\python.exe gen_address_index.py       # 지번→통·반 검색 인덱스
.venv\Scripts\python.exe gen_apartment_index.py     # 아파트 검색 인덱스(아파트 항목 바뀌면)
.venv\Scripts\python.exe gen_issues.py              # 미표시/충돌/미배정 통계
.venv\Scripts\python.exe gen_overview.py            # 전체지도(all_tong/all_admin)
.venv\Scripts\python.exe gen_manifest.py            # 동 목록
```
> 지번 몇 개 고치는 정도면 ②③ 다 돌려도 수십 초~1분대. (매번 앞에 `.venv\Scripts\python.exe`)

**④ 로컬 확인**(선택) — 아래 4번으로 미리보기 후

**⑤ 배포** — 아래 6번(`git push`).

## 6. 배포 (git push)
```powershell
cd "C:\Users\김정순\OneDrive\바탕 화면\shiheung_tongban_map\siheung-gunja-tongban-map"
git add -u                     # 바뀐 추적 파일(관할구역.csv, out/*, index.html 등)
git commit -m "설명"
git push                       # 자동으로 v1/main → Render 배포
```
- **확인:** `git status`가 clean + "ahead" 없음 = 다 올라감. `git push`는 upstream(v1/main)으로 감.
- **origin으로 밀지 말 것**(옛 저장소). 평소 `git push`만 하면 v1로 가니 신경 안 써도 됨.
- 새 파일(문서·이미지 등)은 `git add -u` 대신 파일명을 직접 `git add`.

## 7. 다른 PC에서 이어받을 때(환경 재설치)
저장소만 clone하면 **SHP·.venv가 없음**(gitignore). 아래가 추가로 필요:
1. **연속지적도 SHP** — `LSMD_CONT_LDREG_41390_202606.shp`(+ .dbf/.shx/.prj/.cpg) 를 저장소 루트에 둠. (국가공간정보포털/브이월드의 시흥 41390 연속지적도)
2. **파이썬 환경**
   ```powershell
   python -m venv .venv
   .venv\Scripts\python.exe -m pip install geopandas shapely pandas openpyxl qrcode pillow
   ```
   (SHP 없이도 **앱 미리보기·배포는 가능**. SHP·venv는 "데이터 재빌드"에만 필요.)
